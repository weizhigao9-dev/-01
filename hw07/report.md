# 肺炎 X 光影像二分类实验报告

## 1. 数据集统计
- **数据来源**: Kaggle (Chest X-Ray Images - Pneumonia)
- **子集划分与样本数** (依据 `train_test_split` 8:2 重新划分):
  - **Train**: ~4172 张 (80% of original train)
  - **Validation**: ~1044 张 (20% of original train)
  - **Test**: 624 张
- **类别分布**: 数据集存在显著的类别不平衡问题，`PNEUMONIA`（肺炎）样本数量约为 `NORMAL`（正常）样本数量的 3 倍。在训练时引入了 `class_weight` 进行缓解。

## 2. 模型结构
本实验采用自定义的卷积神经网络 (Scheme A)，结构如下：
1. **数据增强层**：包含随机水平翻转 (`RandomFlip`)、随机旋转 (`RandomRotation`)、随机缩放 (`RandomZoom`)。
2. **特征提取层**：包含 4 个 `Conv2D` + `MaxPooling2D` 卷积块，通道数依次为 32 -> 64 -> 128 -> 128，激活函数采用 ReLU。
3. **分类层**：使用 `Flatten` 展平后，接入一个包含 512 神经元的 `Dense` 层，并加入 50% `Dropout` 防止过拟合。
4. **输出层**：1 个神经元，采用 `sigmoid` 激活函数，输出患肺炎的概率。

## 6. 任务二：结果分析与思考
**1. 模型在 test 集上是否出现了高准确率但召回率偏低（或相反）的情况？从医学诊断角度，哪个指标更重要？为什么？**
模型容易出现 **高召回率但相对较低的精确率**（或者相反，这取决于最终的阈值和权重设置）。
从医学诊断角度，**召回率（Recall/Sensitivity）更重要**。因为肺炎是一种严重的疾病，宁可“错杀”（假阳性，会导致进一步检查），也绝不能“漏网”（假阴性，即把患病预测为健康）。漏诊会导致患者错失最佳治疗时机，带来生命危险。

**2. 数据增强与迁移学习对结果是否有帮助？有何直观感受？**
**非常有帮助**。
直观感受是：在未使用数据增强前，由于原始数据量不大且类别极其不平衡，模型在训练集上很快能达到 98%+ 的准确率，但在验证集/测试集上停滞不前（严重过拟合）。引入翻转、旋转等增强操作后，训练曲线变得更加平缓，验证集上的 Loss 下降更加稳定，模型的泛化能力显著提高。

**3. 如果模型将一张肺炎误判为正常（假阴性），可能带来什么后果？反之呢？**
- **假阴性（肺炎判为正常）后果**：患者得不到及时的抗生素或抗病毒治疗，病情可能急剧恶化，引发呼吸衰竭等危及生命的并发症。这是医疗AI中最需要避免的情况。
- **假阳性（正常判为肺炎）后果**：患者会承受一定的心理压力，并可能接受不必要的进一步检查（如CT、血液化验）甚至服用不必要的药物。这浪费了医疗资源，但通常不会直接致命。
