## 任务三：开源语音识别 (ASR) 调研与实现

### 1. 调研与对比

我选取了目前主流的三种开源 ASR 方案进行对比：**OpenAI Whisper**、**Vosk** 和 **FunASR**。

| 对比维度 | OpenAI Whisper | Vosk | FunASR (阿里) |
| :--- | :--- | :--- | :--- |
| **许可协议** | MIT License | Apache 2.0 | MIT / Apache 2.0 |
| **语言与方言支持** | 99种语言，多语种支持极佳 | 20+种语言，轻量级支持好 | 专注中文及中英混读，支持多种方言 |
| **模型体量与速度** | 体量大 (Base~Large)，无 GPU 推理慢 | 极小 (50MB级别)，CPU 推理极快 | 中等，推理速度较快 (支持批处理) |
| **流式/实时支持** | 原生**不支持**流式 (需第三方封装) | 原生**支持**流式，极低延迟 | **支持** (通过特定模型如 Paraformer-online) |
| **部署难度** | 极低 (`pip install openai-whisper`) | 极低 (提供多语言轻量级 API) | 中等 (需依赖 ModelScope 体系) |
| **PC 实测感受** | 识别准确率惊人，连标点符号都能精准预测，但离线 CPU 运行较大模型时有明显卡顿感。 | 速度飞快，几乎不占资源，适合实时麦克风收音，但对于复杂专业词汇的准确率不如 Whisper。 | 中文识别天花板，对带口音的中文包容度高，部署时下载依赖相对较多。 |

### 2. 选型理由
本项目最终选择 **OpenAI Whisper (Base 模型)** 进行本地代码实现。
**理由**：本次实验的核心需求是识别【任务二】生成的一段完整的录音文件。Whisper 部署极为简便（仅需 pip 安装），且在处理文件级别的长音频时，其上下文理解能力和标点符号自动添加功能远超其他传统模型，能输出非常规范的书面文本。
